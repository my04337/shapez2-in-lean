# 用語集 (Glossary)

Shapez2 で使用される主要な用語をナレッジパネルのカテゴリ別にまとめた用語集です。

---

## Shapes (図形)

### Basic Elements (基本要素)

| 名称 | 説明 |
|---|---|
| Shape (シェイプ・形) | ゲームの主要リソース。形状と色の組み合わせで構成され、ハブに納品することでリサーチを進める。 |
| Color (カラー・色) | シェイプに着色される色情報。クォーターごとに個別に保持される。 |
| Layer (レイヤー・層) | シェイプを垂直方向に積み重ねる単位。ゲームモードにも寄るが最大4層または5層まで積み重ね可能。 |
| Quarters (象限) | シェイプを4等分した各区画。それぞれ独立した形状種別と色を持つ。 |
| Empty Quarters (空の象限) | シェイプが存在しない象限。シェイプコードでは `--` と表記する。 |

### Shape Types (シェイプの種類)

| 名称 | 説明 |
|---|---|
| Circle (サークル) | 円形のシェイプ種別。シェイプコード記号: `C` |
| Rectangle (レクタングル) | 四角形のシェイプ種別。シェイプコード記号: `R` |
| Star (スター) | 星形のシェイプ種別。シェイプコード記号: `S` |
| Windmill (ウィンドミル) | 風車形のシェイプ種別。シェイプコード記号: `W` |
| Pin (ピン) | クォーターを支持するための特殊なシェイプ種別。Support Pin として機能し、スタック時に隙間を埋める土台として使う。シェイプコード記号: `P` |
| Crystal (クリスタル) | 液剤を硬化させて生成される結晶形のシェイプ種別。**Fragile (脆弱)** であり、落下や切断で破損する。シェイプコード記号: `c` |

### Color Types (カラーの種類)

| 名称 | 説明 |
|---|---|
| Red (レッド) | 原色の赤。シェイプコード記号: `r` |
| Green (グリーン) | 原色の緑。シェイプコード記号: `g` |
| Blue (ブルー) | 原色の青。シェイプコード記号: `b` |
| Yellow (イエロー) | レッドとグリーンを混色して生成される色。シェイプコード記号: `y` |
| Cyan (サイアン) | グリーンとブルーを混色して生成される色。シェイプコード記号: `c` |
| Magenta (マゼンタ) | レッドとブルーを混色して生成される色。シェイプコード記号: `m` |
| White (ホワイト) | 全ての色を混色して生成される色。シェイプコード記号: `w` |
| Uncolored (アンカラード) | 色が塗られていない状態。シェイプコード記号: `u` |

### Shape Code Notation (シェイプコード記法)

| 名称 | 説明 |
|---|---|
| Shape Code (シェイプコード) | シェイプの構造を文字列で表現する記法。各象限のパーツコードとカラーコードの組を連結し、複数層は `:` で区切って記述する。右上の象限を起点に時計回りの順で記述し、複数層の場合は下層から上層に向かって記述する。例: `RgRgRgRg` は全象限が緑の四角形の1レイヤーシェイプ。 |
| Part Code (パーツコード) | 象限のシェイプ種別を示す1文字のコード。`C`(サークル), `R`(レクタングル), `S`(スター), `W`(ウィンドミル), `P`(ピン), `c`(クリスタル)。空の象限ではカラーコードと合わせて `--` と表記する。 |
| Color Code (カラーコード) | 象限の色を示す1文字のコード。`r`(レッド), `g`(グリーン), `b`(ブルー), `c`(サイアン), `m`(マゼンタ), `y`(イエロー), `w`(ホワイト), `u`(アンカラード)。ピンのようにカラーコードを持たない種別は `-` を用いる(例: `P-`)。なお `c` はパーツコード(クリスタル)とカラーコード(サイアン)の両方で使用される。 |
| Layer Separator (レイヤー区切り) | シェイプコードでレイヤーを区切る記号 `:` 。例: `CuCuCuCu:RrRrRrRr` は2レイヤーのシェイプ。 |

## Basic Controls (基本操作)

| 名称 | 説明 |
|---|---|
| Vortex (渦) | ゲームの中心施設。指定されたシェイプを納品することでリサーチが進む。**Point Symmetry (点対称)** な図形は向きに関わらず納品が受け付けられる。 |
| Trash (ゴミ箱) | ベルト上のシェイプを削除する建物。不要なシェイプを破棄するために使う。 |

---

## Conveyors & Lifts (コンベアとリフト)

| 名称 | 説明 |
|---|---|
| Belt (ベルト) | シェイプを一方向に搬送するコンベアベルト。 |
| Pipe (パイプ) | 液剤(Color Fluid)を輸送するために必要な管路。輸送は即時行われるが、毎分7,200リットルというスループット制限がある。 |
| Belt Launcher (ベルトランチャー) | ベルト上のシェイプを地下経由で送出する建物。Shapez1 のトンネル入口に相当する。 |
| Belt Receiver (ベルトレシーバー) | ベルトランチャーから送出されたシェイプを受け取る建物。Shapez1 のトンネル出口に相当する。 |
| Pipe Launcher (パイプランチャー) | パイプ上のカラーを地下経由で送出する建物。パイプ版のランチャー。 |
| Pipe Receiver (パイプレシーバー) | パイプランチャーから送出されたカラーを受け取る建物。パイプ版のレシーバー。 |

## Cutting (切断)

| 名称 | 説明 |
|---|---|
| Half-Destroyer (切断処理機) | アイソレーターレーザーでシェイプの **West Half (西側の半分)** を消去し、残りのパーツを出力する建物。 |
| Cutter (切断機) | ホログラフィックブレードでシェイプを2つに切断し、**West Half (西側の半分)** と **East Half (東側の半分)** を並列に出力する建物。 |
| Swapper (スワップ機) | 2つのホログラフィックブレードで2つのシェイプを同時に切断し、それぞれの **West Half (西側の半分)** を入れ替えて出力する建物。 |
| West Half (西側の半分) | シェイプを左右に分割したときの西側（左側）の半分。 |
| East Half (東側の半分) | シェイプを左右に分割したときの東側（右側）の半分。 |
| Cardinal Directions (方角) | Shapez2 における絶対方角。切断やスワップの位置は、建物の設置方向やカメラの視点に依存せず、常に方角により決定される。 |

---

## Rotating (回転)

| 名称 | 説明 |
|---|---|
| Rotator (回転機) | シェイプを時計回り90度単位で回転させる建物。反時計回り・180度回転のバリアントもアンロック可能。 |
| Rotation (回転) | シェイプの回転状態。建物に入る際の回転は加工結果に影響する。 |

---

## Stacking (積み重ね)

| 名称 | 説明 |
|---|---|
| Stacking (積み重ね) | 積み重ねにより、複数の **Layers (レイヤー)** を持つシェイプの製造や、異なる **Quarters (象限)** を1層にまとめることができる。 |
| Stacker (積層機) | 2つのフロアからシェイプを受け取る建物。上のシェイプが下のシェイプの上に落とされ、切断されたパーツは固い面に当たるまで落下する。 |
| Stack (積み重ね) | 2つのシェイプを上下に積み重ねる操作。上のシェイプは下のシェイプの上に落下し、固い面に当たるまで落ちる。 |
| Unstack (積み解体) | 積み重なったシェイプを最上位レイヤーとそれ以下のレイヤーに分離する操作。 |
| Layer Limit (レイヤー制限) | シェイプが持てる最大レイヤー数。標準難易度では4、Insane難易度では5。超過した分のレイヤーは廃棄される。 |

---

## Fluids & Painting (液剤と着色)

| 名称 | 説明 |
|---|---|
| Color Fluids (色付き液剤) | シェイプを着色するために必要な液体。複数の新しいコンセプトを導入する。 |
| Fluid Miner (液剤採掘機) | 液剤を採掘する施設。Pump を設置することで液剤を抽出できる。 |
| Pump (ポンプ) | Fluid Miner に設置する建物。各ポンプは毎分一定量のリットルを抽出する。 |
| Painter (着色機) | 液剤とシェイプの両方を接続して着色する建物。常にシェイプの **Topmost (最上位)** レイヤーを着色する。 |
| Topmost (最上位) | 積み重なったシェイプの一番上のレイヤー。着色機が着色する対象レイヤー。 |

---

## Color Mixing (混色)

| 名称 | 説明 |
|---|---|
| Mixing Colors (混色する) | 2つの Color Fluids を混合する操作。**Secondary (二次色)** や **Tertiary (三次色)** を生成できる。 |
| Secondary (二次色) | 2つの原色を混合して生成される色。Yellow, Cyan, Magenta がこれに当たる。 |
| Tertiary (三次色) | 3つの原色またはさらなる混合によって生成される色。White がこれに当たる。 |
| Color Mixer (混色機) | 2つの Color Fluids を接続して混合する建物。 |
| Fluid Volume (液量) | 液剤の体積。混色時に保存される。例: 赤10リットルと緑10リットルを混ぜると黄20リットルになる。 |
| Additive Color Mixing (加法混色) | デフォルトの色モード。光と同様に機能し、赤・緑・青を混ぜると白になる。 |

## Pin Pusher (ピン押し機)

| 名称 | 説明 |
|---|---|
| Support Pins (サポートピン) | クォーターを保持し、それ自体も積み重ねられるピン。通常では不可能なシェイプを製造できる。 |
| Pin Pusher (ピン押し機) | 第1レイヤーの **non-empty (空でない)** 各 **Quarter (象限)** の下に **Support Pin (サポートピン)** を押し込む建物。 |

## Crystals (結晶)

| 名称 | 説明 |
|---|---|
| Crystal (クリスタル) | 液剤を硬化させて生成される結晶形のシェイプ種別。[「Shapes (図形)」](#shapes-図形)の「シェイプの種類」も参照。 |
| Crystal Generator (結晶製造機) | 提供された液剤をシェイプの **Gaps (隙間)** や **Pins (ピン)** に充填し、その過程で液剤を結晶に固化する建物。 |
| Gaps (隙間) | シェイプの空の象限部分。Crystal Generator はここに結晶を充填する。 |
| Fragile (脆弱) | 結晶の特性。落下すると破損する。例えば積み重ね時に落下して破れる。 |
| Shatter (砕け散り) | 結晶を切断したときに発生する現象。同じ図形の同一・上下レイヤーにつながるすべての結晶が一斉に破壊される。 |
| Break Crystals (結晶を壊す) | 入れ替え中に意図的に結晶を砕けさせる高度なテクニック。結晶の下に隙間のある図形を作ることができる。Hard および Insane 難易度で必要になる。 |

---

## Wires (ワイヤー)

| 名称 | 説明 |
|---|---|
| Wires (ワイヤー) | 工場を完全に自動化し、動的な工場を構築するためのシステム。 |
| Logical Signals (論理信号) | ワイヤー建物が輸送・処理する信号。Number・ Color・Shape・Empty・Conflict の5種がある。 |
| Number (数値) | 論理信号の一種。整数値 (0, 1, 2, 3, …)。 |
| Empty (空) | 論理信号の一種。「null」とも呼ばれる。**Off** と分類される。 |
| Conflict (コンフリクト) | 論理信号の一種。異なる信号を持つ2つの出力を接続したときに発生する。**Off** と分類される。 |
| On (オン) | Empty および数値 0 以外のすべての信号値。論理ゲートにオンとみなされる。 |
| Off (オフ) | Empty および数値 0。論理ゲートにオフとみなされる。 |
| Belt Filter (ベルトフィルタ) | ワイヤー入力信号に基づいて図形の流れを制御する建物。動的工場構築の不可欠なツール。 |
| Make Anything Machine / MaM (なんでも抽出機) | 任意の図形を読み取り自動的に生産できる動的工場。Wiresを活用した最終目標の一つ。 |
| Randomized Shape (ランダム図形) | オペレーターレベルで導入されるランダムな図形目標。 |
| Operator Shape Receiver (オペレーター図形レシーバー) | 指定チャンネルの現在のランダム図形を受信する建物。 |
| Operator Shape (オペレーター図形) | オペレーターレベルで要求される目標図形。Operator Shape Receiver で取得できる。 |
| Simulated Buildings (シミュレーション設備) | 論理信号に格納された図形を処理できる建物。MaM 構築の中心的要素。 |

## Game Mode Variations (ゲームモードによる差異)

> **注記**: 以下の仕様はゲームモードにより異なる。詳細な形式的定義は将来 Lean で記述する予定。

| 項目 | 標準 (Normal / Hard) | Insane (Crazy) | Mod |
|---|---|---|---|
| 1層あたりの象限数 | 4 | 4 | 任意 (6象限等) |
| 最大レイヤー数 | 4 | 5 | 任意 |
| シェイプ種別・カラー種別 | 本書記載の通り | 本書記載の通り | 新種が追加される可能性あり |
| 加工手段 | 本書記載の通り | 本書記載の通り | 新たな操作が追加される可能性あり |

本用語集は原則として標準モードの仕様に基づいて記述している。

加工操作の実践的な解説は [HOWTO.md](HOWTO.md) を参照。
