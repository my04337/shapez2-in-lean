# 結晶の砕け散り (Crystal Shatter)

Shapez2 における **結晶 (Crystal)** の **砕け散り (Shatter)** 操作に関する厳密な仕様書。

---

## 1. 概要

**砕け散り (Shatter)** とは、**脆弱 (Fragile)** な属性を持つシェイプ種別である **結晶 (Crystal)** が、特定の操作によって破壊される現象である。  
砕け散りが発生すると、対象の結晶は **空 (empty)** に置き換わる。  
結晶は互いに **結合 (Bond)** しうるため、一つの結晶の砕け散りが **結合クラスタ (Bonded Cluster)** 全体に連鎖する。

砕け散りのトリガーは以下の2種類:

| トリガー | 発生条件 |
|---|---|
| **落下 (Fall)** | 落下対象となった脆弱な象限が砕け散り、結合クラスタ全体に伝播する |
| **切断 (Cutting)** | 東西に跨がる結合クラスタが砕け散る |

---

## 2. 脆弱 (Fragile) 属性

**脆弱 (Fragile)** とは、落下や切断で破損しうる性質のことである。  
現在のゲーム仕様において、脆弱な属性を持つ `Quarter` は **結晶 (Crystal)** のみである。

| Quarter コンストラクタ | 脆弱 |
|---|---|
| `Quarter.empty` | いいえ |
| `Quarter.pin` | いいえ |
| `Quarter.crystal color` | **はい** |
| `Quarter.colored part color` | いいえ |

Lean における判定関数:
```
Quarter.isFragile : Quarter → Bool
  | crystal _ => true
  | _         => false
```

---

## 3. 結晶の結合条件 (Crystal Bonding)

結晶同士は特定の条件下で **結合 (Bond)** する。
結合している結晶の集合を **結合クラスタ (Bonded Cluster)** と呼ぶ。
結合は推移的であり、A-B 間に結合があり B-C 間に結合がある場合、{A, B, C} 全体が一つの結合クラスタを形成する。

### 3.1 同レイヤ内の結合条件

同一レイヤ内で、**同じ色** の結晶が **隣接** している場合に結合する。

> 隣接関係の定義（同レイヤ内の隣接ペア、対角ペアの非隣接など）については [`adjacency.md`](adjacency.md) を参照。

#### 同レイヤ結合のルール

象限 A と象限 B が同レイヤ内で結合する条件:
1. A と B が **同レイヤ内で隣接** している（[隣接ペア](adjacency.md#3-同レイヤ内の隣接関係) の4組）
2. A と B がともに `Quarter.crystal` である
3. A と B の **色が同じ** である

#### 同レイヤ結合の例

- `crcrcrcr` — 4象限すべてが赤の結晶 → 全て結合して1クラスタ
- `crcr----` — NE(cr) と SE(cr) が隣接同色 → 結合
- `cr----cr` — NE(cr) と NW(cr) が隣接同色 → 結合
- `crcgcrcg` — NE(cr)≠SE(cg), SE(cg)≠SW(cr), SW(cr)≠NW(cg), NW(cg)≠NE(cr) → 結合なし（4つの独立クラスタ）
- `crcrcgcg` — NE(cr)-SE(cr) が結合、SW(cg)-NW(cg) が結合 → 2つの独立クラスタ

### 3.2 上下レイヤ間の結合条件

上下に隣接するレイヤ間で、**同じ方角** の象限が**ともに結晶**であれば結合する。  
同レイヤ内と異なり、**色は問わない**。

> 上下レイヤ間の隣接（垂直隣接）の定義については [`adjacency.md`](adjacency.md#4-上下レイヤ間の隣接関係垂直隣接) を参照。

#### 上下レイヤ結合のルール

レイヤ L₁ の象限 A と、その直上のレイヤ L₂ の象限 B が上下レイヤ間で結合する条件:
1. A と B が **同じ方角** (NE, SE, SW, NW) に位置する
2. A と B がともに `Quarter.crystal` である
3. L₁ と L₂ が **垂直に隣接** するレイヤである

> **注意**: 異なる方角の象限同士（例: L₁ の NE と L₂ の SE）は上下レイヤ間で結合しない。

#### 上下レイヤ結合の例

| シェイプコード | 結合状態 | 説明 |
|---|---|---|
| `cr------:cg------` | 1クラスタ | L1:NE(cr) と L2:NE(cg) が同方角の結晶 → 色が異なっても結合 |
| `crcrP---:--cgcg--` | 1クラスタ | L1:NE(cr)-SE(cr) が同レイヤ結合、L2:SW(cg)-NW(cg) が同レイヤ結合、L1:SE(cr) と L2:SE(cg) が上下結合 → 全体で1クラスタ |
| `cr------:--cg----` | 2クラスタ | L1:NE(cr) と L2:SE(cg) は **異なる方角** → 結合しない |

## 4. 砕け散りトリガー

### 4.1 落下 (Fall) による砕け散り

**落下** は主に以下の操作で発生する:
- **積み重ね (Stacking)**: 上のシェイプ側が落下対象となる
- **ピン押し (Pin Pushing)**: 破棄された最上位レイヤにしか接続していなかった象限が落下対象となる場合がある

落下は **シェイプ全体** が対象となる場合と、**一部の象限のみ** が対象となる場合がある。

#### 落下による砕け散りのルール

1. 落下対象となった象限のうち、**脆弱 (Fragile)** な象限を列挙する
2. それらの象限が属する **結合クラスタ** を求める
3. 結合クラスタ内の **全ての結晶** を砕け散り対象とする
4. 砕け散り対象の象限を `Quarter.empty` に置き換える

> **注意**: 非脆弱な象限（通常パーツ、ピン、空）は落下しても砕け散らない。

#### 落下による砕け散りの例

| 変化前 | 落下対象 | 変化後 | 説明 |
|---|---|---|---|
| `crcrcrcr` | シェイプ全体 | `--------` (空) | 全結晶が落下対象 → 全砕け |
| `crCr----` | シェイプ全体 | `--Cr----` | NE(cr) のみ脆弱 → NE が砕ける。SE(Cr) は非脆弱なので残る |
| `crcrcgcg` | シェイプ全体 | `--------` (空) | 2クラスタ {NE,SE}, {SW,NW} とも落下対象 → 全砕け |
| `cr------:cg------` | シェイプ全体 | `--------:--------` (空) | L1:NE(cr) と L2:NE(cg) が結合 → クラスタ全体が砕ける |

### 4.2 切断 (Cutting) による砕け散り

**切断** はシェイプを常に **南北方向** で二分する操作であり、以下の建物で発生する:
- **切断処理機 (Half-Destroyer)**
- **切断機 (Cutter)**
- **スワップ機 (Swapper)**

切断により、シェイプは **西側の半分 (West Half)** と **東側の半分 (East Half)** に分かれる
（[East/West Half の定義](adjacency.md#東西の半分-east--west-half) を参照）。

#### 切断による砕け散りのルール

1. シェイプ内の全ての **結合クラスタ** を求める
2. 各クラスタについて、East Half に属する象限と West Half に属する象限の **両方** を含むかを判定する
3. **東西に跨がる** クラスタ（East Half と West Half の両方に象限を持つクラスタ）を砕け散り対象とする
4. 砕け散り対象クラスタ内の **全ての結晶** を `Quarter.empty` に置き換える

> **注意**: East Half のみ、または West Half のみに収まるクラスタは砕け散らない。

#### 切断による砕け散りの例（砕ける場合）

| 変化前 | 変化後 | 説明 |
|---|---|---|
| `crcrcrcr` | `--------` (空) | 全4象限が赤結晶で1クラスタ → 東西に跨がる → 全砕け |
| `crRgSbcr` | `--RgSb--` | NE(cr) と NW(cr) が隣接同色で結合 → 東西に跨がる → cr 2つが砕ける。Rg, Sb は非結晶なので残る |
| `CrcgcgWu` | `Cr----Wu` | SE(cg) と SW(cg) が隣接同色で結合 → 東西に跨がる → cg 2つが砕ける。Cr, Wu は非結晶なので残る |
| `crcrP---:--cgcg--` | `----P---:--------` (L2 は空) | L1:NE(cr)-SE(cr) が同レイヤ結合、L2:SW(cg)-NW(cg) が同レイヤ結合、L1:SE(cr)-L2:SE(cg) が上下結合 → 全結晶が1クラスタ → 東西に跨がる → 全結晶が砕ける。P- はピンなので残る |

#### 切断による砕け散りの例（砕けない場合）

| シェイプコード | 説明 |
|---|---|
| `crcgcrcg` | NE(cr)≠SE(cg), SE(cg)≠SW(cr), SW(cr)≠NW(cg), NW(cg)≠NE(cr) → 4つの独立した結晶。各クラスタは1象限なので東西に跨がらない |
| `crcrcgcg` | {NE(cr), SE(cr)} と {SW(cg), NW(cg)} の2クラスタ。前者は East Half のみ、後者は West Half のみ → 東西に跨がらない |

---

## 5. 砕け散りの効果

砕け散り対象と判定された結晶象限は、**`Quarter.empty` に置き換わる**。  
すなわち、砕け散った結晶は完全に消滅する。

砕け散りは **切断・落下の前処理** として実行される:
- **切断の場合**: 砕け散り → 切断（東西分離）の順
- **落下の場合**: 砕け散り → 落下の順

---

## 6. スコープ制限

本仕様では以下を **範囲外** とする:

- 砕け散り後に発生する **落下** 操作（空中に浮いたままとする）
- **落下** 操作自体の定義・実装
- **切断** 操作自体の定義・実装（東西分離のロジック）
- 結晶製造機 (Crystal Generator) による結晶の生成
