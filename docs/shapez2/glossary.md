# 用語集 (Glossary)

Shapez2 で使用される主要な用語をナレッジパネルのカテゴリ別にまとめた用語集です。

---

## General Concepts (一般概念)

### Game Overview (ゲーム概要)

| 名称 | 説明 |
|---|---|
| Shapez2 (シェイプズ2) | 形状と色の組み合わせを加工して指定された図形を作る工場建設ゲーム。 |
| Delivery (納品) | シェイプを渦に提出すること。指定されたシェイプを納品することでリサーチが進む。 |

### Buildings and Spatial Concepts (施設と空間概念)

| 名称 | 説明 |
|---|---|
| Vortex (渦) | ゲームの中心施設。指定されたシェイプを納品することでリサーチが進む。**Point Symmetry (点対称)** な図形は向きに関わらず納品が受け付けられる。ただし **Line Symmetry (線対称)** のみの図形は点対称ではないため、向きによっては納品が拒否される。 |
| Space Platform (宇宙プラットフォーム) | 渦以外の各種施設を設置するための追加プラットフォーム。 |
| Space View (スペースビュー) | 宇宙プラットフォーム全体を俯瞰したり設置削除を行うビュー。 |
| Machine View (マシンビュー) | 宇宙プラットフォーム内に各種設備などを設置するビュー。 |
| Cardinal Directions (方角) | Shapez2 における絶対方角。切断やスワップの位置は、建物の設置方向やカメラの視点に依存せず、常に方角により決定される。 |
| West Half (西側の半分) | シェイプを左右に分割したときの西側（左側）の半分。 |
| East Half (東側の半分) | シェイプを左右に分割したときの東側（右側）の半分。 |

## Shape (シェイプ)

### Shape Components (シェイプの構成要素)

| 名称 | 説明 |
|---|---|
| Shape (シェイプ・形) | ゲームの主要リソース。形状と色の組み合わせで構成され、渦に納品することでリサーチを進める。 |
| Color (カラー・色) | シェイプに着色される色情報。象限ごとに個別に保持される。 |
| Layer (レイヤ・層) | シェイプを垂直方向に積み重ねる単位。ゲームモードにもよるが最刧4層または5層まで積み重ね可能。 |
| Quarters (象限) | シェイプを4等分した各区画。それぞれ独立した形状種別と色を持つ。 |
| Empty Quarters (空の象限) | シェイプが存在しない象限。シェイプコードでは `--` と表記する。 |

### Shape Types (シェイプの種類)

| 名称 | 説明 |
|---|---|
| Circle (サークル) | 円形のシェイプ種別。シェイプコード記号: `C` |
| Rectangle (レクタングル) | 四角形のシェイプ種別。シェイプコード記号: `R` |
| Star (スター) | 星形のシェイプ種別。シェイプコード記号: `S` |
| Windmill (ウィンドミル) | 風車形のシェイプ種別。シェイプコード記号: `W` |
| Pin (ピン) | 象限を支持するための特殊なシェイプ種別。ゲーム公式名称は **Support Pin (サポートピン)** だが、本プロジェクトでは**ピン**に統一する。スタック時に隙間を埋める土台として使う。シェイプコード記号: `P` |
| Crystal (クリスタル) | 液剤を硬化させて生成される結晶形のシェイプ種別。**Fragile (脆弱)** であり、落下や切断で破損する。シェイプコード記号: `c` |

### Color Types (カラーの種類)

| 名称 | 説明 |
|---|---|
| Red (レッド) | 原色の赤。シェイプコード記号: `r` |
| Green (グリーン) | 原色の緑。シェイプコード記号: `g` |
| Blue (ブルー) | 原色の青。シェイプコード記号: `b` |
| Yellow (イエロー) | レッドとグリーンを混色して生成される色。シェイプコード記号: `y` |
| Cyan (サイアン) | グリーンとブルーを混色して生成される色。シェイプコード記号: `c` |
| Magenta (マゼンタ) | レッドとブルーを混色して生成される色。シェイプコード記号: `m` |
| White (ホワイト) | 全ての色を混色して生成される色。シェイプコード記号: `w` |
| Uncolored (アンカラード) | 色が塗られていない状態。シェイプコード記号: `u` |

### Shape Code Notation (シェイプコード記法)

| 名称 | 説明 |
|---|---|
| Shape Code (シェイプコード) | シェイプの構造を文字列で表現する記法。各象限のパーツコードとカラーコードの組を連結し、複数層は `:` で区切って記述する。右上の象限を起点に時計回りの順で記述し、複数層の場合は下層から上層に向かって記述する。例: `RgRgRgRg` は全象限が緑の四角形の1レイヤシェイプ。 |
| Part Code (パーツコード) | 象限のシェイプ種別を示す1文字のコード。`C`(サークル), `R`(レクタングル), `S`(スター), `W`(ウィンドミル), `P`(ピン), `c`(クリスタル)。空の象限ではカラーコードと合わせて `--` と表記する。 |
| Color Code (カラーコード) | 象限の色を示す1文字のコード。`r`(レッド), `g`(グリーン), `b`(ブルー), `c`(サイアン), `m`(マゼンタ), `y`(イエロー), `w`(ホワイト), `u`(アンカラード)。ピンのようにカラーコードを持たない種別は `-` を用いる(例: `P-`)。なお `c` はパーツコード(クリスタル)とカラーコード(サイアン)の両方で使用される。 |
| Layer Separator (レイヤ区切り) | シェイプコードでレイヤを区切る記号 `:` 。例: `CuCuCuCu:RrRrRrRr` は2レイヤのシェイプ。 |

## Shape Processing (シェイプ加工)

### Cutting (切断)
シェイプを切断する操作。  
切断されたパーツは固い面に当たるまで落下する。切断の位置は建物の設置方向やカメラの視点に依存せず、常に **Cardinal Directions (方角)** により決定される。  
シェイプを **Rotator (回転機)** で回転させることで、どちら側が West Half になるかを制御できる。  
「切断 → 回転 → 切断」の操作により、シェイプを **Quarters (象限)** 単位に分解できる。

| 名称 | 説明 |
|---|---|
| Half-Destroyer (切断処理機) | アイソレーターレーザーでシェイプの **West Half (西側の半分)** を消去し、残りのパーツを出力する建物。 |
| Cutter (切断機) | ホログラフィックブレードでシェイプを2つに切断し、**West Half (西側の半分)** と **East Half (東側の半分)** を並列に出力する建物。 |
| Swapper (スワップ機) | 2つのホログラフィックブレードで2つのシェイプを同時に切断し、それぞれの **West Half (西側の半分)** を入れ替えて出力する建物。 |

### Rotating (回転)
シェイプを回転する操作。  
回転によってどのパーツがどこに落ちるかを制御できるため、切断した半分を意図した位置に積み重ねるなどの応用が可能。

| 名称 | 説明 |
|---|---|
| Rotator (回転機) | シェイプを時計回り90度単位で回転させる建物。 |
| Reverse Rotator (逆回転機) | シェイプを反時計回り90度単位で回転させる建物。 |
| 180° Rotator (180度回転機) | シェイプを180度回転させる建物。 |

### Stacking (積み重ね)
シェイプを積み重ねる操作。  
上のシェイプは下のシェイプの上に落下し、固い面に当たるまで落ちる。  
両方のシェイプが完全な場合は互いの上に重なる。象限が空または欠落している場合、一部のパーツはさらに落下することがある。  
積み重ねにより、複数のレイヤを持つシェイプの製造や、異なる象限ㅠ1層にまとめることができる。  
シェイプが持てる最大レイヤ数には限りがある。標準難易度では4、Insane 難易度では5。超過した分のレイヤは廃棄される。

| 名称 | 説明 |
|---|---|
| Stacker (積層機) | 2つのフロアからシェイプを受け取る建物。上のシェイプが下のシェイプの上に落とされ、切断されたパーツは固い面に当たるまで落下する。 |

### Painting (着色)
シェイプを着色する操作。  
最上位レイヤのみが着色され、下位レイヤは着色されない。  
これは最上位レイヤに空の象限がある場合でも同様で、空の象限やその下のレイヤは着色されない。

| 名称 | 説明 |
|---|---|
| Painter (着色機) | 液剤とシェイプの両方を接続して着色する建物。常にシェイプの **Topmost (最上位)** レイヤを着色する。 |


### Color Mixing (混色)
複数の液剤(Color Fluids)を混合して新しい色を生成する操作。  
混色された色は、混色前の液剤の体積の合計を持つ。例えば赤10リットルと緑10リットルを混ぜると黄20リットルになる。  
Shapez2 ではデフォルトで加法混色が使用されている。  

| 名称 | 説明 |
|---|---|
| Color Mixer (混色機) | 2つの液剤を接続して混合する建物。 |

#### Color Classification (混色の種類)
Shapez2 での色の分類は以下の通り。
- Primary Colors (原色): Red, Green, Blue
- Secondary Colors (二次色): Yellow, Cyan, Magenta
- Tertiary Colors (三次色): White

#### Mixing Rules (混色のルール)
混色の基本的なルールは以下の通り。
両者に共通の原色が無い場合、混色の結果は両方の色を含む色になる。例えば、RedとGreenを混ぜるとYellowになる。
両者に共通の原色がある場合、混色の結果は両方の色から共通の原色を除いた色になる。例えば、Cyan と Magenta を混ぜると Blue になる。

| 色1 | 色2 | 結果の色 |
|---|---|---|
| Red | Red | Red |
| Red | Green | Yellow |
| Red | Blue | Magenta |
| Green | Red | Yellow |
| Green | Green | Green |
| Green | Blue | Cyan |
| Blue | Red | Magenta |
| Blue | Green | Cyan |
| Blue | Blue | Blue |
| Yellow | Red | Red |
| Yellow | Green | Green |
| Yellow | Blue | Blue |
| Cyan | Red | Red |
| Cyan | Green | Green |
| Cyan | Blue | Blue |
| Magenta | Red | Red |
| Magenta | Green | Green |
| Magenta | Blue | Blue |
| White | Red | Red |
| White | Green | Green |
| White | Blue | Blue |
| White | Yellow | Yellow |
| White | Cyan | Cyan |
| White | Magenta | Magenta |
| White | White | White |

### Pin Pushing (ピン押し)
シェイプを支持するためのピンを押し込む操作。
接続されていない象限は積み重ね時に落下する。  
Pin Pusher は第1レイヤの空でない各象限の下にピンを押し込む。ピンは一つの象限としても振る舞い、それ自体も積み重ねられるため、通常では不可能な大きな隙間のあるシェイプや複数レイヤにわたるシェイプを製造できる。  
Pin Pusher でシェイプを押し上げてレイヤ上限を超えると、超過分が廃棄される。この性質を逆利用することで、ピンを最上位に配置したシェイプを作ることもできる。

| 名称 | 説明 |
|---|---|
| Pin Pusher (ピン押し機) | 第1レイヤの **non-empty (空でない)** 各 **Quarter (象限)** の下にピンを押し込む建物。 |


### Crystals (結晶)
シェイプを液剤で硬化させて生成される結晶形のシェイプ種別。  
結晶は **Fragile (脆弱)** であり、落下や切断で破損する。  
落下や切断をすると **Shatter (砕け散り)** が発生し、同じシェイプの同一・上下レイヤでつながるすべての結晶が一斉に破壊される。  
積み重ねによる破壊は避けられないが、切断しない限り **Swapper (スワップ機)** で結晶の半分を入れ替えることはできる。  
入れ替え中に意図的に **Break Crystals (結晶を壊す)** ことで、結晶の下に隙間のあるシェイプを作ることができる。Hard および Insane 難易度で必要になる。

| 名称 | 説明 |
|---|---|
| Crystal Generator (結晶製造機) | 提供された液剤をシェイプの **Gaps (隙間)** や **Pins (ピン)** に充填し、その過程で液剤を結晶に固化する建物。複数の結晶製造機を連続させることで、異なる色の結晶ㅠ1つのシェイプ内に組み合わせることができる。 |

### Trash (廃棄)
シェイプを削除する操作。  
不要なシェイプを破棄したりスループットの測定などの目的で使用される。

| 名称 | 説明 |
|---|---|
| Trash (ゴミ箱) | ベルト上のシェイプを削除する建物。不要なシェイプを破棄するために使う。 |

## Wires and Logic (ワイヤーと論理演算)

**Wire (ワイヤー)** は論理信号を輸送するための建物で、シェイプや色などの情報を信号として伝える。ワイヤーに関連する建物は、フロー制御（Belt Filter, Pipe Gate など）、論理ゲート、シミュレーション装置に分類される。

### Logic Signals (論理信号)
ワイヤー建物は **Logical Signals (論理信号)** を輸送・処理する。信号は次の5種類:
- Number (数値): 整数値 (0, 1, 2, …)
- Color (色): シェイプの色を表す信号。例: color-r, color-g, color-b など。
- シェイプコード (Shape Code): シェイプの構造を表す信号。例: CuCuCuCu など。
- Empty (空): 「null」とも呼ばれる。**Off** と分類される。
- Conflict (コンフリクト): 異なる信号を持つ2出力を接続したときに発生する。**Off** と分類される。

Empty と数値 0 は **Off** 、それ以外はすべて **On** に分類される。

### Wire Basics (ワイヤーの基礎)

| 名称 | 説明 |
|---|---|
| Signal Producer (信号生成器) | 任意のシェイプコードや色信号、論理信号を生成する建物。 |
| Belt Filter (ベルトフィルター) | ベルト上のシェイプを特定の条件でフィルタリングする建物。入力信号に応じた動作は下表を参照。 |
| Pipe Gate (パイプゲート) | パイプ内の液剤の流れを制御する建物。特定の条件で液剤の流れを On / Off することができる。 |

#### Belt Filter Behavior (ベルトフィルターの動作)

| 入力信号 | 動作 |
|---|---|
| Shape | 一致するシェイプは通過、不一致は横に除外 |
| Number (0以外) | 通過。0 の場合は除外 |
| Empty / Conflict | ベルトを完全に遮断 |

#### Pipe Gate Behavior (パイプゲートの動作)

| 入力信号 | 動作 |
|---|---|
| Color | 一致する色の液剤は通過、不一致は遮断 |
| Number (0以外) | 通過。0 の場合は遮断 |
| Empty / Conflict | パイプを完全に遮断 |


### Logic Gates (論理ゲート)
基本的な論理演算を行う建物。例えば、AND Gate は複数の入力がすべて On の場合にのみ出力が On になる。

| 名称 | 説明 |
|---|---|
| AND Gate (ANDゲート) | すべての入力が On の場合にのみ出力が On になる建物。 |
| OR Gate (ORゲート) | いずれかの入力が On の場合に出力が On になる建物。 |
| NOT Gate (NOTゲート) | 入力が On の場合に出力が Off、入力が Off の場合に出力が On になる建物。 |
| XOR Gate (XORゲート) | 入力のうちちょうど１つが On の場合に出力が On になる建物。 |
| Comparison Gate (比較ゲート) | 2つの入力を比較し、条件に応じて出力を On / Off する建物。 |

比較ゲートの条件には下記6種類がある。

| 条件 | 説明 |
|---|---|
| Equal (等しい) | 2つの入力が等しい場合に出力が On になる。 |
| Not Equal (等しくない) | 2つの入力が等しくない場合に出力が On になる。 |
| Greater (より大きい) | 入力Aが入力Bより大きい場合に出力が On になる。 |
| GreaterEqual (以上) | 入力Aが入力B以上の場合に出力が On になる。 |
| Less (より小さい) | 入力Aが入力Bより小さい場合に出力が On になる。 |
| LessEqual (以下) | 入力Aが入力B以下の場合に出力が On になる。 |

### Simulated Machines (シミュレーション装置)
論理信号に格納されたシェイプの処理やシミュレーションを行う装置。

| 名称 | 説明 |
|---|---|
| Simulated Rotator (シミュレーション回転機) | 入力されたシェイプコードを90°回転させて出力する建物。 |
| Simulated Reverse Rotator (シミュレーション逆回転機) | 入力されたシェイプコードを反時計回りに90°回転させて出力する建物。 |
| Simulated Half-Destroyer (シミュレーション切断処理機) | 入力されたシェイプコードの **West Half (西側の半分)** を削除し、残りのシェイプコードを出力する建物。 |
| Simulated Swapper (シミュレーションスワップ機) | 入力された2つのシェイプコードを同時に切断し、それぞれの **West Half (西側の半分)** を入れ替えて出力する建物。 |
| Simulated Stacker (シミュレーション積層機) | 2つの入力されたシェイプコードを積み重ねて出力する建物。 |
| Simulated Unstacker (シミュレーション積み重ね解除機) | 入力されたシェイプコードの最上位レイヤを分割して2つのシェイプコードとして出力する建物。 |
| Simulated Painter (シミュレーション着色機) | 入力されたシェイプコードの最上位レイヤを着色して出力する建物。 |
| Simulated Pin Pusher (シミュレーションピン押し機) | 入力されたシェイプコードの第1レイヤの空でない各象限の下にピンを押し込んだシェイプコードを出力する建物。 |

### Make Anything Machine / MaM (全自動工場)

オペレーターレベルでは **Randomized Shape (ランダム図形)** 目標が導入される。**Operator Shape Receiver (オペレーター図形レシーバー)** で目標図形を論理信号として取得し、任意のシェイプを自動生産する **Make Anything Machine (MaM)** を構築することが推奨される。

**Simulated Buildings (シミュレーション設備)** を使うと、論理信号内のシェイプに対して切断・回転などの処理を仮想的に実行できる。切断処理機・回転機・切断処理機の順で処理を繰り返すことで、シェイプ信号を指定象限のみに絞り込める。これを全象限 × 全シェイプタイプに対して行い結果を積み重ねることで、任意の無着色１レイヤのシェイプを製造できる。
