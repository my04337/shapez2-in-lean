# 用語集 (Glossary)

Shapez2 で使用される主要な用語をナレッジパネルのカテゴリ別にまとめた用語集です。

---

## 一般概念 (General Concepts)

### ゲーム概要 (Game Overview)

| 名称 | 説明 |
|---|---|
| Shapez2 | 固有名詞。形状と色の組み合わせを加工して指定された図形を作る工場建設ゲーム。 |
| 納品 (Delivery) | シェイプを渦に提出すること。指定されたシェイプを納品することでリサーチが進む。 |

### 施設と空間概念 (Buildings and Spatial Concepts)

| 名称 | 説明 |
|---|---|
| 渦 (Vortex) | ゲームの中心施設。指定されたシェイプを納品することでリサーチが進む。**点対称 (Point Symmetry)** な図形は向きに関わらず納品が受け付けられる。ただし **線対称 (Line Symmetry)** のみの図形は点対称ではないため、向きによっては納品が拒否される。 |
| 宇宙プラットフォーム (Space Platform) | 渦以外の各種施設を設置するための追加プラットフォーム。 |
| スペースビュー (Space View) | 宇宙プラットフォーム全体を俯瞰したり設置削除を行うビュー。 |
| マシンビュー (Machine View) | 宇宙プラットフォーム内に各種設備などを設置するビュー。 |
| 方角 (Cardinal Directions) | Shapez2 における絶対方角。切断やスワップの位置は、建物の設置方向やカメラの視点に依存せず、常に方角により決定される。 |
| 西側の半分 (West Half) | シェイプを左右に分割したときの西側（左側）の半分。 |
| 東側の半分 (East Half) | シェイプを左右に分割したときの東側（右側）の半分。 |

## シェイプ (Shape)

### シェイプの構成要素 (Shape Components)

| 名称 | 説明 |
|---|---|
| シェイプ・形 (Shape) | ゲームの主要リソース。形状と色の組み合わせで構成され、渦に納品することでリサーチを進める。 |
| カラー・色 (Color) | シェイプに着色される色情報。象限ごとに個別に保持される。 |
| レイヤ・層 (Layer) | シェイプを垂直方向に積み重ねる単位。ゲームモードにもよるが最大4層または5層まで積み重ね可能。 |
| 象限 (Quarters) | シェイプを4等分した各区画。それぞれ独立した形状種別と色を持つ。 |
| 空の象限 (Empty Quarters) | シェイプが存在しない象限。シェイプコードでは `--` と表記する。 |

### シェイプの種類 (Shape Types)

| 名称 | 説明 |
|---|---|
| サークル (Circle) | 円形のシェイプ種別。シェイプコード記号: `C` |
| レクタングル (Rectangle) | 四角形のシェイプ種別。シェイプコード記号: `R` |
| スター (Star) | 星形のシェイプ種別。シェイプコード記号: `S` |
| ウィンドミル (Windmill) | 風車形のシェイプ種別。シェイプコード記号: `W` |
| ピン (Pin) | 象限を支持するための特殊なシェイプ種別。ゲーム公式名称は **サポートピン (Support Pin)** だが、本プロジェクトでは**ピン**に統一する。スタック時に隙間を埋める土台として使う。シェイプコード記号: `P` |
| クリスタル (Crystal) | 液剤を硬化させて生成される結晶形のシェイプ種別。**脆弱 (Fragile)** であり、落下や切断で破損する。シェイプコード記号: `c` |

### カラーの種類 (Color Types)

| 名称 | 説明 |
|---|---|
| レッド (Red) | 原色の赤。シェイプコード記号: `r` |
| グリーン (Green) | 原色の緑。シェイプコード記号: `g` |
| ブルー (Blue) | 原色の青。シェイプコード記号: `b` |
| イエロー (Yellow) | レッドとグリーンを混色して生成される色。シェイプコード記号: `y` |
| シアン (Cyan) | グリーンとブルーを混色して生成される色。シェイプコード記号: `c` |
| マゼンタ (Magenta) | レッドとブルーを混色して生成される色。シェイプコード記号: `m` |
| ホワイト (White) | 全ての色を混色して生成される色。シェイプコード記号: `w` |
| アンカラード (Uncolored) | 色が塗られていない状態。シェイプコード記号: `u` |

### シェイプコード記法 (Shape Code Notation)

| 名称 | 説明 |
|---|---|
| シェイプコード (Shape Code) | シェイプの構造を文字列で表現する記法。各象限のパーツコードとカラーコードの組を連結し、複数層は `:` で区切って記述する。右上の象限を起点に時計回りの順で記述し、複数層の場合は下層から上層に向かって記述する。例: `RgRgRgRg` は全象限が緑の四角形の1レイヤシェイプ。 |
| パーツコード (Part Code) | 象限のシェイプ種別を示す1文字のコード。`C`(サークル), `R`(レクタングル), `S`(スター), `W`(ウィンドミル), `P`(ピン), `c`(クリスタル)。空の象限ではカラーコードと合わせて `--` と表記する。 |
| カラーコード (Color Code) | 象限の色を示す1文字のコード。`r`(レッド), `g`(グリーン), `b`(ブルー), `c`(シアン), `m`(マゼンタ), `y`(イエロー), `w`(ホワイト), `u`(アンカラード)。ピンのようにカラーコードを持たない種別は `-` を用いる(例: `P-`)。なお `c` はパーツコード(クリスタル)とカラーコード(シアン)の両方で使用される。 |
| レイヤ区切り (Layer Separator) | シェイプコードでレイヤを区切る記号 `:` 。例: `CuCuCuCu:RrRrRrRr` は2レイヤのシェイプ。 |

## シェイプ加工 (Shape Processing)

### 切断 (Cutting)
シェイプを切断する操作。  
切断されたパーツは固い面に当たるまで落下する。切断の位置は建物の設置方向やカメラの視点に依存せず、常に **方角 (Cardinal Directions)** により決定される。  
シェイプを **回転機 (Rotator)** で回転させることで、どちら側が West Half になるかを制御できる。  
「切断 → 回転 → 切断」の操作により、シェイプを **象限 (Quarters)** 単位に分解できる。

| 名称 | 説明 |
|---|---|
| 切断処理機 (Half-Destroyer) | アイソレーターレーザーでシェイプの **西側の半分 (West Half)** を消去し、残りのパーツを出力する建物。 |
| 切断機 (Cutter) | ホログラフィックブレードでシェイプを2つに切断し、**西側の半分 (West Half)** と **東側の半分 (East Half)** を並列に出力する建物。 |
| スワップ機 (Swapper) | 2つのホログラフィックブレードで2つのシェイプを同時に切断し、それぞれの **西側の半分 (West Half)** を入れ替えて出力する建物。 |

### 回転 (Rotating)
シェイプを回転する操作。  
回転によってどのパーツがどこに落ちるかを制御できるため、切断した半分を意図した位置に積み重ねるなどの応用が可能。

#### 回転方向の定義 (Rotation Direction)

| 用語 | 説明 |
|---|---|
| +方向の回転 (Positive Rotation / CW) | 表示上の **時計回り (Clockwise, CW)** に相当する回転方向。本プロジェクトでは回転量を正の値で表す場合、この方向を指す。 |
| −方向の回転 (Negative Rotation / CCW) | 表示上の **反時計回り (Counter-Clockwise, CCW)** に相当する回転方向。 |

シェイプコードにおける象限の並び順は「右上 (NE) を起点に時計回り」（NE → SE → SW → NW）である。  
+90°（時計回り）の回転を適用すると、各象限は **コード上で右方向に1位置分ローテート** される。

- 例: `Cr------` の場合、象限0(NE)が `Cr`、残りは空。  
  +90°回転後 → `--Cr----`（象限1(SE)に移動）

これは方角の変換にも対応する。例えば象限0(NE)にあるパーツを+90°回転させると象限1(SE)の位置に移動する（北東 → 南東）。

| 名称 | 説明 |
|---|---|
| 回転機 (Rotator) | シェイプを時計回り90度単位で回転させる建物。 |
| 逆回転機 (Reverse Rotator) | シェイプを反時計回り90度単位で回転させる建物。 |
| 180度回転機 (180° Rotator) | シェイプを180度回転させる建物。 |

### 積み重ね (Stacking)
シェイプを積み重ねる操作。  
上のシェイプは下のシェイプの上に落下し、固い面に当たるまで落ちる。  
両方のシェイプが完全な場合は互いの上に重なる。象限が空または欠落している場合、一部のパーツはさらに落下することがある。  
積み重ねにより、複数のレイヤを持つシェイプの製造や、異なる象限を1層にまとめることができる。  
シェイプが持てる最大レイヤ数には限りがある。標準難易度では4、Insane 難易度では5。超過した分のレイヤは廃棄される。

| 名称 | 説明 |
|---|---|
| 積層機 (Stacker) | 2つのフロアからシェイプを受け取る建物。上のシェイプが下のシェイプの上に落とされ、切断されたパーツは固い面に当たるまで落下する。 |

### 着色 (Painting)
シェイプを着色する操作。  
最上位レイヤのみが着色され、下位レイヤは着色されない。  
これは最上位レイヤに空の象限がある場合でも同様で、空の象限やその下のレイヤは着色されない。

| 名称 | 説明 |
|---|---|
| 着色機 (Painter) | 液剤とシェイプの両方を接続して着色する建物。常にシェイプの **最上位 (Topmost)** レイヤを着色する。 |


### 混色 (Color Mixing)
複数の液剤(Color Fluids)を混合して新しい色を生成する操作。  
混色された色は、混色前の液剤の体積の合計を持つ。例えば赤10リットルと緑10リットルを混ぜると黄20リットルになる。  
Shapez2 ではデフォルトで加法混色が使用されている。  

| 名称 | 説明 |
|---|---|
| 混色機 (Color Mixer) | 2つの液剤を接続して混合する建物。 |

#### 混色の種類 (Color Classification)
Shapez2 での色の分類は以下の通り。
- 原色 (Primary Colors): Red, Green, Blue
- 二次色 (Secondary Colors): Yellow, Cyan, Magenta
- 三次色 (Tertiary Colors): White

#### 混色のルール (Mixing Rules)
混色の基本的なルールは以下の通り。
両者に共通の原色が無い場合、混色の結果は両方の色を含む色になる。例えば、RedとGreenを混ぜるとYellowになる。
両者に共通の原色がある場合、混色の結果は両方の色から共通の原色を除いた色になる。例えば、Cyan と Magenta を混ぜると Blue になる。

| 色1 | 色2 | 結果の色 |
|---|---|---|
| Red | Red | Red |
| Red | Green | Yellow |
| Red | Blue | Magenta |
| Green | Red | Yellow |
| Green | Green | Green |
| Green | Blue | Cyan |
| Blue | Red | Magenta |
| Blue | Green | Cyan |
| Blue | Blue | Blue |
| Yellow | Red | Red |
| Yellow | Green | Green |
| Yellow | Blue | Blue |
| Cyan | Red | Red |
| Cyan | Green | Green |
| Cyan | Blue | Blue |
| Magenta | Red | Red |
| Magenta | Green | Green |
| Magenta | Blue | Blue |
| White | Red | Red |
| White | Green | Green |
| White | Blue | Blue |
| White | Yellow | Yellow |
| White | Cyan | Cyan |
| White | Magenta | Magenta |
| White | White | White |

### ピン押し (Pin Pushing)
シェイプを支持するためのピンを押し込む操作。
接続されていない象限は積み重ね時に落下する。  
Pin Pusher は第1レイヤの空でない各象限の下にピンを押し込む。ピンは一つの象限としても振る舞い、それ自体も積み重ねられるため、通常では不可能な大きな隙間のあるシェイプや複数レイヤにわたるシェイプを製造できる。  
Pin Pusher でシェイプを押し上げてレイヤ上限を超えると、超過分が廃棄される。この性質を逆利用することで、ピンを最上位に配置したシェイプを作ることもできる。

| 名称 | 説明 |
|---|---|
| ピン押し機 (Pin Pusher) | 第1レイヤの **空でない (non-empty)** 各 **象限 (Quarter)** の下にピンを押し込む建物。 |


### 結晶 (Crystals)
シェイプを液剤で硬化させて生成される結晶形のシェイプ種別。  
結晶は **脆弱 (Fragile)** であり、落下や切断で破損する。  
落下や切断をすると **砕け散り (Shatter)** が発生し、同じシェイプの同一・上下レイヤでつながるすべての結晶が一斉に破壊される。  
積み重ねによる破壊は避けられないが、切断しない限り **スワップ機 (Swapper)** で結晶の半分を入れ替えることはできる。  
入れ替え中に意図的に **結晶を壊す (Break Crystals)** ことで、結晶の下に隙間のあるシェイプを作ることができる。Hard および Insane 難易度で必要になる。

| 名称 | 説明 |
|---|---|
| 結晶製造機 (Crystal Generator) | 提供された液剤をシェイプの **隙間 (Gaps)** や **ピン (Pins)** に充填し、その過程で液剤を結晶に固化する建物。複数の結晶製造機を連続させることで、異なる色の結晶を1つのシェイプ内に組み合わせることができる。 |

### 廃棄 (Trash)
シェイプを削除する操作。  
不要なシェイプを破棄したりスループットの測定などの目的で使用される。

| 名称 | 説明 |
|---|---|
| ゴミ箱 (Trash) | ベルト上のシェイプを削除する建物。不要なシェイプを破棄するために使う。 |

## ワイヤーと論理演算 (Wires and Logic)

**ワイヤー (Wire)** は論理信号を輸送するための建物で、シェイプや色などの情報を信号として伝える。ワイヤーに関連する建物は、フロー制御（Belt Filter, Pipe Gate など）、論理ゲート、シミュレーション装置に分類される。

### 論理信号 (Logic Signals)
ワイヤー建物は **論理信号 (Logical Signals)** を輸送・処理する。信号は次の5種類:
- 数値 (Number): 整数値 (0, 1, 2, …)
- 色 (Color): シェイプの色を表す信号。例: color-r, color-g, color-b など。
- シェイプコード (Shape Code): シェイプの構造を表す信号。例: CuCuCuCu など。
- 空 (Empty): 「null」とも呼ばれる。**Off** と分類される。
- コンフリクト (Conflict): 異なる信号を持つ2出力を接続したときに発生する。**Off** と分類される。

Empty と数値 0 は **Off** 、それ以外はすべて **On** に分類される。

### ワイヤーの基礎 (Wire Basics)

| 名称 | 説明 |
|---|---|
| 信号生成器 (Signal Producer) | 任意のシェイプコードや色信号、論理信号を生成する建物。 |
| ベルトフィルター (Belt Filter) | ベルト上のシェイプを特定の条件でフィルタリングする建物。入力信号に応じた動作は下表を参照。 |
| パイプゲート (Pipe Gate) | パイプ内の液剤の流れを制御する建物。特定の条件で液剤の流れを On / Off することができる。 |

#### ベルトフィルターの動作 (Belt Filter Behavior)

| 入力信号 | 動作 |
|---|---|
| Shape | 一致するシェイプは通過、不一致は横に除外 |
| Number (0以外) | 通過。0 の場合は除外 |
| Empty / Conflict | ベルトを完全に遮断 |

#### パイプゲートの動作 (Pipe Gate Behavior)

| 入力信号 | 動作 |
|---|---|
| Color | 一致する色の液剤は通過、不一致は遮断 |
| Number (0以外) | 通過。0 の場合は遮断 |
| Empty / Conflict | パイプを完全に遮断 |


### 論理ゲート (Logic Gates)
基本的な論理演算を行う建物。例えば、AND Gate は複数の入力がすべて On の場合にのみ出力が On になる。

| 名称 | 説明 |
|---|---|
| ANDゲート (AND Gate) | すべての入力が On の場合にのみ出力が On になる建物。 |
| ORゲート (OR Gate) | いずれかの入力が On の場合に出力が On になる建物。 |
| NOTゲート (NOT Gate) | 入力が On の場合に出力が Off、入力が Off の場合に出力が On になる建物。 |
| XORゲート (XOR Gate) | 入力のうちちょうど１つが On の場合に出力が On になる建物。 |
| 比較ゲート (Comparison Gate) | 2つの入力を比較し、条件に応じて出力を On / Off する建物。 |

比較ゲートの条件には下記6種類がある。

| 条件 | 説明 |
|---|---|
| 等しい (Equal) | 2つの入力が等しい場合に出力が On になる。 |
| 等しくない (Not Equal) | 2つの入力が等しくない場合に出力が On になる。 |
| より大きい (Greater) | 入力Aが入力Bより大きい場合に出力が On になる。 |
| 以上 (GreaterEqual) | 入力Aが入力B以上の場合に出力が On になる。 |
| より小さい (Less) | 入力Aが入力Bより小さい場合に出力が On になる。 |
| 以下 (LessEqual) | 入力Aが入力B以下の場合に出力が On になる。 |

### シミュレーション装置 (Simulated Machines)
論理信号に格納されたシェイプの処理やシミュレーションを行う装置。

| 名称 | 説明 |
|---|---|
| シミュレーション回転機 (Simulated Rotator) | 入力されたシェイプコードを90°回転させて出力する建物。 |
| シミュレーション逆回転機 (Simulated Reverse Rotator) | 入力されたシェイプコードを反時計回りに90°回転させて出力する建物。 |
| シミュレーション切断処理機 (Simulated Half-Destroyer) | 入力されたシェイプコードの **西側の半分 (West Half)** を削除し、残りのシェイプコードを出力する建物。 |
| シミュレーションスワップ機 (Simulated Swapper) | 入力された2つのシェイプコードを同時に切断し、それぞれの **西側の半分 (West Half)** を入れ替えて出力する建物。 |
| シミュレーション積層機 (Simulated Stacker) | 2つの入力されたシェイプコードを積み重ねて出力する建物。 |
| シミュレーション積み重ね解除機 (Simulated Unstacker) | 入力されたシェイプコードの最上位レイヤを分割して2つのシェイプコードとして出力する建物。 |
| シミュレーション着色機 (Simulated Painter) | 入力されたシェイプコードの最上位レイヤを着色して出力する建物。 |
| シミュレーションピン押し機 (Simulated Pin Pusher) | 入力されたシェイプコードの第1レイヤの空でない各象限の下にピンを押し込んだシェイプコードを出力する建物。 |
| シミュレーション結晶製造機 (Simulated Crystal Generator) | 入力されたシェイプコードの隙間やピンを結晶で埋めたシェイプコードを出力する建物。 |

### 全自動工場 (Make Anything Machine / MaM)

オペレーターレベルでは **ランダム図形 (Randomized Shape)** 目標が導入される。**オペレーター図形レシーバー (Operator Shape Receiver)** で目標図形を論理信号として取得し、任意のシェイプを自動生産する **Make Anything Machine (MaM)** を構築することが推奨される。

**シミュレーション設備 (Simulated Buildings)** を使うと、論理信号内のシェイプに対して切断・回転などの処理を仮想的に実行できる。切断処理機・回転機・切断処理機の順で処理を繰り返すことで、シェイプ信号を指定象限のみに絞り込める。これを全象限 × 全シェイプタイプに対して行い結果を積み重ねることで、任意の無着色１レイヤのシェイプを製造できる。
