# 初学者のための数学用語簡易辞書

本ドキュメントは、Lean 4 ガイド群に登場する高度な数学用語を、高校数学（数III の微分積分・数学C の行列）程度の知識を持つ読者向けに簡潔に解説するものです。

---

## 論理学の基礎

### 命題（Proposition）
真か偽のいずれかが定まる文のこと。「3 は素数である」は命題（真）、「x は偶数である」は x の値が決まらない限り命題ではなく**述語**（後述）。Lean 4 では `Prop` 型で表現される。

### 述語（Predicate）
変数を含み、変数に具体値を代入すると命題になるもの。例: 「n は偶数である」は自然数 n に関する述語。Lean 4 では `Nat → Prop` のように関数として表現される。

### 論理結合子（Logical Connective）
命題同士を組み合わせる演算子。

| 記号 | 名称 | 意味 |
|------|------|------|
| `∧` | 論理積（conjunction） | P かつ Q |
| `∨` | 論理和（disjunction） | P または Q |
| `¬` | 否定（negation） | P でない |
| `→` | 含意（implication） | P ならば Q |
| `↔` | 同値（if and only if） | P と Q は同値 |

### 全称量化（Universal Quantification）
「すべての x について P(x) が成り立つ」ことを表す。記号は `∀`。例: `∀ n : Nat, n + 0 = n`。
→ [Wikipedia: 全称記号](https://ja.wikipedia.org/wiki/%E5%85%A8%E7%A7%B0%E8%A8%98%E5%8F%B7)

### 存在量化（Existential Quantification）
「ある x が存在して P(x) が成り立つ」ことを表す。記号は `∃`。例: `∃ n : Nat, n * n = 4`。
→ [Wikipedia: 存在記号](https://ja.wikipedia.org/wiki/%E5%AD%98%E5%9C%A8%E8%A8%98%E5%8F%B7)

### 排中律（Law of Excluded Middle）
任意の命題 P について「P であるか、P でないか」のいずれかが成り立つという原理。古典論理では公理として認められるが、構成的論理（後述）では認められない。Lean 4 では `Classical.em` として利用可能。
→ [Wikipedia: 排中律](https://ja.wikipedia.org/wiki/%E6%8E%92%E4%B8%AD%E5%BE%8B)

## 型理論・依存型理論

### 型理論（Type Theory）
集合論に代わる数学の基礎付けの枠組み。すべてのオブジェクトに「型」を割り当て、型の規則によってプログラムや証明の正しさを保証する。Lean 4 の基盤。
→ [Wikipedia: 型理論](https://ja.wikipedia.org/wiki/%E5%9E%8B%E7%90%86%E8%AB%96)

### 依存型（Dependent Type）
値に依存して決まる型。例えば「長さ n のベクトル」型は自然数 n の値によって異なる型になる。Lean 4 では `(n : Nat) → Vector α n` のように書く。通常の関数型 `A → B` は、戻り値の型が引数に依存しない特殊ケース。
→ [Wikipedia: 依存型](https://ja.wikipedia.org/wiki/%E4%BE%9D%E5%AD%98%E5%9E%8B)

### Calculus of Constructions（CoC）
依存型理論の一種。Lean 4 のカーネルはこの拡張版（Calculus of Inductive Constructions）に基づく。命題論理・述語論理・高階論理をすべて統一的に扱える。

### 宇宙（Universe）と宇宙階層（Universe Hierarchy）
「型の型」を表す概念。Lean 4 では `Prop`（命題の宇宙）、`Type 0`（データの宇宙）、`Type 1`（`Type 0` の型）、…と階層をなす。「すべての型を含む型」を仮定するとラッセルのパラドックスが生じるため、階層が必要になる。

### カリー＝ハワード同型対応（Curry-Howard Correspondence）
「命題＝型」「証明＝プログラム」という対応関係。例えば「A ならば B」という命題は関数型 `A → B` に対応し、その証明は A を受け取って B を返す関数に対応する。Lean 4 はこの対応を根幹に置いた言語。
→ [Wikipedia: カリー＝ハワード同型対応](https://ja.wikipedia.org/wiki/%E3%82%AB%E3%83%AA%E3%83%BC%EF%BC%9D%E3%83%8F%E3%83%AF%E3%83%BC%E3%83%89%E5%90%8C%E5%9E%8B%E5%AF%BE%E5%BF%9C)

## 帰納型・再帰

### 帰納型（Inductive Type）
有限個のコンストラクタ（構築子）によって定義される型。自然数 `Nat` は `zero | succ (n : Nat)` という2つのコンストラクタで定義される帰納型の典型例。リスト・二分木なども帰納型として定義できる。

### 帰納族（Inductive Family）
型パラメータがコンストラクタごとに異なる値を取れる帰納型。「長さ n のベクトル」型が代表例で、`nil` は長さ 0、`cons` は長さ n+1 の型を返す。

### 構造的再帰（Structural Recursion）
再帰呼び出しのたびに引数が「構造的に小さくなる」ことで停止性を保証する再帰の手法。Lean 4 の `def` で最もよく使われるパターン。例: `Nat.succ n` に対して `n` は構造的に小さい。

### 整礎再帰（Well-Founded Recursion）
整礎関係（well-founded relation、無限降下列が存在しない順序関係）に基づいて停止性を保証する再帰。構造的再帰では表現できない複雑な再帰パターンに使われる。Lean 4 では `termination_by` で停止測度を指定する。

### 整礎関係（Well-Founded Relation）
どんな空でない部分集合にも極小元が存在するような順序関係。自然数上の `<` が典型例。整礎再帰の基盤。

## 公理系と論理体系

### 構成的論理（Constructive Logic）
「P または Q」を証明するには P か Q のどちらか一方を実際に構成（exhibit）しなければならないとする論理体系。排中律を公理として認めない。Lean 4 のカーネルは構成的だが、`Classical` モジュールで古典論理も使える。

### 古典論理（Classical Logic）
排中律を公理として認める論理体系。「背理法」による非構成的な存在証明が可能。日常的な数学の多くはこちらに基づく。

### 選択公理（Axiom of Choice）
任意の空でない集合の族に対し、各集合から1つずつ元を選ぶ関数（選択関数）が存在するという公理。Lean 4 では `Classical.choice` として利用可能。古典的な数学では広く使われるが、構成的数学では認められない。
→ [Wikipedia: 選択公理](https://ja.wikipedia.org/wiki/%E9%81%B8%E6%8A%9E%E5%85%AC%E7%90%86)

### 命題の外延性（Propositional Extensionality, propext）
同値な命題は等しいという公理。すなわち `(P ↔ Q) → (P = Q)`。Lean 4 に組み込まれた公理の一つ。

### 関数の外延性（Function Extensionality, funext）
すべての入力に対して同じ出力を返す2つの関数は等しいという原理。すなわち `(∀ x, f x = g x) → f = g`。Lean 4 では `funext` タクティクで利用する。

### 商型（Quotient Type）
同値関係で「同じとみなした」元を一つにまとめて作る型。例えば整数は「自然数のペア (a, b) を a - b が等しいもの同士で同一視」して商型として構成できる。Lean 4 のカーネルに組み込まれている。

## 代数的構造

### モノイド（Monoid）
結合法則を満たす二項演算と単位元を持つ代数的構造。例: 自然数と加算（単位元は 0）、文字列と連結（単位元は空文字列）。Lean 4 では `Monoid` 型クラスで定義される。

### 群（Group）
モノイドに逆元の存在を加えた構造。例: 整数と加算（逆元は符号反転）。

### 可換環（Commutative Ring）
加算と乗算の2つの演算を持ち、加算は可換群、乗算はモノイド、分配法則が成り立ち、乗算も可換な構造。例: 整数全体の集合。高校数学で扱う整数・有理数・実数の四則演算の構造を抽象化したもの。

### 体（Field）
可換環に、0 以外の元の乗法逆元（いわゆる「割り算」）が存在するという条件を加えたもの。例: 有理数、実数、複素数。

## 同値関係と性質

### 同値関係（Equivalence Relation）
反射律・対称律・推移律の3つを全て満たす二項関係。

| 性質 | 定義 | 例（`=` の場合） |
|------|------|------|
| 反射律（Reflexivity） | `∀ a, a ∼ a` | `a = a` |
| 対称律（Symmetry） | `∀ a b, a ∼ b → b ∼ a` | `a = b → b = a` |
| 推移律（Transitivity） | `∀ a b c, a ∼ b → b ∼ c → a ∼ c` | `a = b → b = c → a = c` |

### 合同関係 / 合同閉包（Congruence / Congruence Closure）
等式に関して「関数を適用しても等しさが保たれる」性質。`a = b → f a = f b` のこと。Lean 4 では `congrArg` や `congr` タクティクで使う。

## 関数型プログラミングの概念

### カリー化（Currying）
複数引数の関数を、1引数ずつ受け取る関数の連鎖に変換すること。例: `f(x, y) = x + y` を `g(x) = (fun y => x + y)` に変換。Lean 4 では関数は常にカリー化された形で定義される。
→ [Wikipedia: カリー化](https://ja.wikipedia.org/wiki/%E3%82%AB%E3%83%AA%E3%83%BC%E5%8C%96)

### 高階関数（Higher-Order Function）
関数を引数としてとる、あるいは関数を返す関数。例: `List.map` は「変換関数」を受け取ってリストの各要素に適用する。

### 多相性 / ポリモーフィズム（Polymorphism）
型をパラメータとして受け取ることで、異なる型に対して同じコードを適用可能にする仕組み。例: `List.length` は `List α` の長さを返し、`α` が何であっても動作する。

### ファンクタ（Functor）
「写像で構造を保つ変換」を抽象化した概念。プログラミングでは `map` 操作を持つコンテナ型。`List`、`Option`、`IO` などが典型例。圏論由来の概念だが、Lean 4 では `Functor` 型クラスとして `map : (α → β) → F α → F β` のインターフェースと理解すればよい。

### モナド（Monad）
ファンクタの拡張で、「計算の逐次的な合成」を抽象化した概念。`bind`（`>>=`）操作を提供する。`Option`（失敗する可能性のある計算）、`IO`（副作用を伴う計算）、`List`（非決定的計算）などが典型例。`do` 記法で直感的に使える。

### 型クラス（Type Class）
型に対して特定の操作やインスタンスを後付けで定義する仕組み。多相関数に「この型は比較できる」「この型は足し算できる」等の制約を表現するために使う。Lean 4 における型クラスは Haskell の型クラスに似ている。

## その他の重要概念

### 決定可能性（Decidability）
命題 P に対し、P が真か偽かをアルゴリズム的に判定できること。Lean 4 では `Decidable` 型クラスで表現され、`if ... then ... else ...` を使うには当該命題が `Decidable` でなければならない。

### 型強制（Coercion）
ある型から別の型へ自動的に変換する仕組み。例えば自然数 `Nat` を整数 `Int` として自動的に扱えるようにする。Lean 4 では `Coe` 型クラスで定義する。

### Strict Positivity（厳格正値性）
帰納型の定義において、定義中の型自身が関数の引数の型に現れてはならないという制約。例えば `inductive Bad where | mk : (Bad → Nat) → Bad` は禁止される。この制約はカーネルの論理的健全性を保証するために必要。

### Subsingleton Elimination（部分単元型の除去）
帰納型の値が最大1つしか存在しない（部分単元型である）場合に、`Prop` の値から `Type` の値を抽出できるという規則。`Prop` は通常プログラムの計算に使えないが、例えば `True` や `∀ a b : α, a = b` のような `Prop` は例外的にデータ抽出に利用できる。
