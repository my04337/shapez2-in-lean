# タクティク詳細ページ — 品質チェック基準

> このファイルは詳細ページの品質基準を定義する。新規作成・レビュー時に参照すること。

---

## 必須セクション

各詳細ページには以下のセクションを**すべて**含めること:

| # | セクション | 必須条件 |
|---|---|---|
| 1 | ヘッダ行（`# タクティク名 — 説明`） | カテゴリ・定義元・ソースを明記 |
| 2 | 概要 | 2–4 文。そのタクティクが「何をするか」を正確に記述 |
| 3 | ゴールパターン | 適用前/適用後の Lean コードブロックで具体例を示す |
| 4 | 構文 | 全バリアントを Lean コードブロックで記載 |
| 5 | 必須 import | Mathlib タクティクの場合のみ。Lean4 core は「組み込み」と明記 |
| 6 | オプション一覧 | テーブル形式。オプションがない場合は「特になし」と明記 |
| 7 | Pros（使うべき場面） | 箇条書き 2–4 項目 |
| 8 | Cons（注意・失敗ケース） | 箇条書き 2–4 項目 |
| 9 | コードサンプル | 2–3 例。各例にゴール状態のコメントを含む |
| 10 | 関連タクティク | テーブル形式（タクティク / 関係 / 使い分け） |
| 11 | 参照 | 公式ドキュメントへのリンク |
| 12 | 出典フッタ | `> **出典**: ...` 形式で末尾に記載 |

---

## 粒度の統一基準

- ファイル行数: 60–130 行（これを大幅に逸脱する場合は調整）
- コードサンプル数: 2–4 例
- Pros/Cons: 各 2–4 項目
- 関連タクティク: 3–6 行

---

## 事実正確性チェック

- [ ] コードサンプルの Lean 4 構文が正しいこと
- [ ] 定義元モジュールパスが正確であること
- [ ] `import` 文が正しいこと（Lean4 core を Mathlib と誤記していないか等）
- [ ] バージョン依存の記載がある場合、対応バージョンが正確であること
- [ ] Pros/Cons が事実に基づいており、誇張・過小評価がないこと

---

## フォーマット統一

- 出典フッタは `TEMPLATE.md` の「出典フッタ規則」に従う
- ヘッダの基本情報行: `**カテゴリ**: TAG | **定義元**: \`path\` | **ソース**: Lean4 core / Mathlib`
