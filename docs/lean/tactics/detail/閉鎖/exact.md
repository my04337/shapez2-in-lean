# `exact` — 項でゴールを正確に閉じる

**カテゴリ**: 閉鎖 | **定義元**: `Lean.Parser.Tactic.exact` | **ソース**: Lean4 core

---

## 概要

`exact` は、ゴールの型と完全に一致する項を直接与えて証明を閉じるタクティクである。
仮定やコンストラクタ、関数適用など任意の項を渡すことができ、型が一致しない場合はエラーとなる。
最も基本的かつ頻繁に使われるゴール閉鎖タクティクであり、他のタクティクの最終ステップとしても多用される。

---

## ゴールパターン

**適用前**
```lean
h : P
⊢ P
```

**適用後**
```lean
-- ゴールなし（証明完了）
```

---

## 構文

```lean
exact term
```

`term` はゴールの型と一致する任意の式。

---

## 必須 import

Lean4 core 組み込みのため不要。

---

## オプション一覧

| 構文 | 意味 |
|---|---|
| `exact e` | 項 `e` でゴールを閉じる |
| `exact ?_` | プレースホルダを残してゴールの型を確認する（デバッグ用） |
| `exact show P from e` | 型注釈付きで項を渡す |

---

## Pros（使うべき場面）

- 仮定がゴールの型と完全一致するとき、最も直接的に証明を閉じられる
- コンストラクタや関数適用を直接渡すことで、複数ステップを 1 行にまとめられる
- 型が明確なので、証明の意図が読み手に伝わりやすい

---

## Cons（注意・失敗ケース）

- 型が完全一致しないとエラーになる（暗黙の型変換は限定的）
- ユニフィケーション不能な場合、`exact` ではなく `apply` + サブゴール分割が必要
- 長大な項を直接書くと可読性が下がるため、`have` や `let` で中間項を導入する方がよい場合がある

---

## コードサンプル

### サンプル 1: 仮定をそのまま渡す

```lean
example (h : 2 + 3 = 5) : 2 + 3 = 5 := by
  -- ⊢ 2 + 3 = 5
  exact h
  -- ゴールなし（証明完了）
```

### サンプル 2: コンストラクタを渡す

```lean
example : True := by
  -- ⊢ True
  exact True.intro
  -- ゴールなし（証明完了）
```

### サンプル 3: 関数適用で閉じる

```lean
example (h : P → Q) (hp : P) : Q := by
  -- ⊢ Q
  exact h hp
  -- ゴールなし（証明完了）
```

## 関連タクティク

| タクティク | 関係 | 使い分け |
|---|---|---|
| `apply` | 類似 | `apply` は関数の結論部でマッチし、引数をサブゴールに分割する。`exact` は完全な項を一度に渡す |
| `refine` | 拡張 | `refine` は `?_` プレースホルダを含む不完全な項を渡し、残りをサブゴールにする |
| `assumption` | 特殊化 | `assumption` は仮定を自動検索して `exact` する。仮定名が不要な場合に便利 |
