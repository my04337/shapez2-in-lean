# `have` — ローカルコンテキストに仮定・中間命題を追加

**カテゴリ**: 補題適用 | **定義元**: `Lean.Parser.Tactic.tacticHave_` | **ソース**: Lean4 core

---

## 概要

`have` は証明中のローカルコンテキストに新しい仮定（不透明な定義）を追加するタクティクである。
追加する命題の型を宣言し、その証明をサブゴールとして生成する。
証明が完了すると、元のゴールの仮定リストに新しい項が加わる。
`let` と異なり、追加される項は**不透明**（内部の定義が展開されない）である。
複雑な証明を中間ステップに分割するための最も基本的な手段。

---

## ゴールパターン

**適用対象**: 任意（ゴールの型に制約なし）

**適用前**
```lean
⊢ P
```

**適用後** (`have h : Q := ...` または `have h : Q := by ...`)
```lean
h : Q
⊢ P
```

---

## 構文

```lean
have h : T := e              -- 項を直接与える
have h : T := by tac          -- タクティクで証明する
have h := e                   -- 型を推論に任せる
have : T := e                 -- 名前を省略（`this` に束縛）
have ⟨a, b⟩ := e             -- パターンマッチで分解
have h : T := by              -- ブロック内で証明
  tac₁
  tac₂
```

---

## 必須 import

Lean4 core 組み込みのため不要。

---

## オプション一覧

| 構文 | 意味 |
|---|---|
| `have h : T := e` | 名前と型を明示して項を与える |
| `have h := e` | 型推論に任せる |
| `have : T := e` | 名前省略（`this` として参照可能） |
| `have ⟨a, b⟩ := e` | 構造体・積をパターンで分解 |
| `have h : T := by ...` | タクティクブロックで証明 |

---

## Pros（使うべき場面）

- 複雑なゴールを中間命題に分割し、証明を段階的に進められる
- 導入した仮定は後続のすべてのタクティクで利用可能
- パターンマッチにより、存在命題や積型の分解を `have ⟨a, b⟩ := ...` で簡潔に行える
- 証明の可読性が向上し、レビューやメンテナンスが容易になる

---

## Cons（注意・失敗ケース）

- 同名の仮定が既にある場合、古い仮定がシャドウされる（消去はされない）
- 不透明なので、`have h := 42` とした場合 `h` の定義本体は `simp` 等で展開されない
- 不要な `have` の乱用は証明を冗長にする。自明なステップは `exact` や `apply` で直接閉じるべき
- `have` で導入した仮定を使い忘れると、未使用の仮定が残る

---

## コードサンプル

### サンプル 1: 中間命題の導入

```lean
example (a b : Nat) (ha : a > 0) (hb : b > 0) : a + b > 0 := by
  -- ⊢ a + b > 0
  have h : a + b ≥ 1 := by omega
  -- h : a + b ≥ 1
  -- ⊢ a + b > 0
  omega
```

### サンプル 2: 存在命題の分解

```lean
example (h : ∃ n : Nat, n > 5) : ∃ n : Nat, n > 3 := by
  -- ⊢ ∃ n, n > 3
  have ⟨n, hn⟩ := h
  -- n : Nat
  -- hn : n > 5
  -- ⊢ ∃ n, n > 3
  exact ⟨n, by omega⟩
```

### サンプル 3: 名前省略で `this` を活用

```lean
example (f : Nat → Nat) (hf : ∀ x, f x = x + 1) : f 3 = 4 := by
  -- ⊢ f 3 = 4
  have : f 3 = 3 + 1 := hf 3
  -- this : f 3 = 3 + 1
  -- ⊢ f 3 = 4
  omega
```

## 関連タクティク

| タクティク | 関係 | 使い分け |
|---|---|---|
| `let` | 透明版 | `let` は透明な定義を導入。`simp` 等で展開される |
| `suffices` | 逆方向 | 後で証明する命題を先に使う。`have` の逆順 |
| `obtain` | パターン分解版 | `have ⟨a, b⟩ := ...` のシンタックスシュガー |
| `replace` | 上書き版 | 同名の既存仮定を置き換える。`have` はシャドウのみ |
