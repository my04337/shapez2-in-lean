# `refine` — `exact` 類似だがプレースホルダ `?_` でサブゴール生成

**カテゴリ**: 補題適用 | **定義元**: `Lean.Parser.Tactic.refine` | **ソース**: Lean4 core

---

## 概要

`refine` は、不完全な証明項を与えてゴールを部分的に埋めるタクティクである。
項中に `?_` プレースホルダを置くと、その部分が新しいサブゴールとして生成される。
`exact` と異なり証明を一度に完成させる必要がなく、`apply` と異なり項の構造を明示的に制御できるため、
複雑な型を持つゴールの証明を段階的に組み立てる場面で特に有用である。

---

## ゴールパターン

**適用対象**: 仮定依存（ゴールの型に応じた項を構築する）

**適用前**
```lean
⊢ A ∧ B
```

**適用後** (`refine ⟨?_, ?_⟩`)
```lean
-- ゴール 1: ⊢ A
-- ゴール 2: ⊢ B
```

---

## 構文

```lean
refine e                 -- ?_ を含む項を与える
refine ⟨?_, ?_⟩         -- 匿名コンストラクタ＋プレースホルダ
refine ⟨a, ?_, rfl⟩     -- 一部を確定、残りをサブゴール化
refine ?_                -- ゴールの型を確認するだけ（デバッグ用）
```

---

## 必須 import

Lean4 core 組み込みのため不要。

---

## オプション一覧

| 構文 | 意味 |
|---|---|
| `refine e` | 明示的な `?_` のみサブゴール化 |
| `refine' e` | 暗黙の `_` もサブゴール化（旧式、互換性のため残存） |
| `refine ?_` | ゴールの型を変更せず確認（`exact ?_` と同様） |

---

## Pros（使うべき場面）

- 複合型（`∧`, `∃`, `Σ` 等）のゴールを分割しつつ、一部を即座に埋められる
- `apply` では制御しにくい引数の順序や位置を、項の構造で明示できる
- `exact` では一度に書ききれない長い証明項を段階的に構築できる
- `?_` の位置で意図的にサブゴールを生成できるため、証明の流れが読みやすい

---

## Cons（注意・失敗ケース）

- `?_` の個数がコンストラクタの引数と合わないとエラーになる
- `refine'` との混同に注意。`refine` は明示 `?_` のみ、`refine'` は `_` も対象
- 暗黙引数を手動で埋める必要がある場合、`@refine` や `(show T from ?_)` が必要になることがある
- 項全体の型推論が失敗するとユニフィケーションエラーが発生する

---

## コードサンプル

### サンプル 1: 存在命題の証明

```lean
example : ∃ n : Nat, n + 1 = 3 := by
  -- ⊢ ∃ n, n + 1 = 3
  refine ⟨2, ?_⟩
  -- ⊢ 2 + 1 = 3
  rfl
```

### サンプル 2: 連言の分割と部分適用

```lean
example (ha : 1 > 0) : 1 > 0 ∧ 2 > 0 := by
  -- ⊢ 1 > 0 ∧ 2 > 0
  refine ⟨ha, ?_⟩
  -- ⊢ 2 > 0
  omega
```

### サンプル 3: 関数適用で引数を部分指定

```lean
example (h : ∀ x y : Nat, x < y → x ≤ y) : 3 ≤ 5 := by
  -- ⊢ 3 ≤ 5
  refine h 3 5 ?_
  -- ⊢ 3 < 5
  omega
```

## バリアントと派生形

| バリアント | 概要 |
|---|---|
| [`refine'`](refine'.md) | 全ての `_` をサブゴール化する旧版 refine |

## 関連タクティク| `exact` | 厳密版 | ゴールを完全に閉じる項を要求。プレースホルダ不可 |
| `apply` | 逆方向推論 | 結論部でマッチし前提をサブゴール化。項構造の制御は限定的 |
| `have` | 中間命題導入 | 新しい仮定を導入して後続で利用。`refine` は既存ゴールを直接変形 |
| `constructor` | コンストラクタ適用 | `refine ⟨?_, ?_⟩` の省略形に相当（帰納型のとき） |
