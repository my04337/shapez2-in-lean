# `skip` — 何もしないタクティク

**カテゴリ**: 制御 | **定義元**: `Lean.Parser.Tactic.skip` | **ソース**: Lean4 core

---

## 概要

`skip` は何もせず常に成功するタクティクである。ゴールに一切変化を与えない。プレースホルダーやメタプログラミングでの no-op として使用する。

---

## ゴールパターン

**適用前**
```lean
⊢ P
```

**適用後**: 変化なし（常に成功）

---

## 構文

```lean
skip
```

---

## 必須 import

Lean4 core 組み込みのため不要。

---

## オプション一覧

特になし。

---

## Pros（使うべき場面）

- タクティクブロックが空だとエラーになる場所のプレースホルダー
- メタプログラミングでの条件分岐で「何もしない」分岐
- `try skip` のような慣用句で他のタクティクのテスト

---

## Cons（注意・失敗ケース）

- ゴールを閉じないため、実質的な証明の進展はない
- 最終証明に `skip` が残っていると `done` で失敗する
- `sorry` と混同しないこと（`skip` はゴールを閉じない）

---

## コードサンプル

### サンプル 1: プレースホルダー

```lean
example : True := by
  skip
  trivial
```

### サンプル 2: try との組み合わせ

```lean
example : 1 = 1 := by
  try skip  -- 常に成功するが何もしない
  rfl
```

## 関連タクティク

| タクティク | 関係 | 使い分け |
|---|---|---|
| `fail` | 常に失敗 | 失敗を明示するなら `fail` |
| `sorry` | ゴール閉鎖 | ゴールを仮閉じするなら `sorry` |
| `trivial` | 簡易閉鎖 | ゴールを閉じるなら `trivial` |
