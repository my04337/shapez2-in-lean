# `show` — ゴールの型を明示して対象ゴールを選択する

**カテゴリ**: 変換 | **定義元**: `Lean.Parser.Tactic.show` | **ソース**: Lean4 core

---

## 概要

`show` はゴールの型を明示的に指定するタクティクである。
複数のゴールがある場合、指定した型にユニファイできる最初のゴールをメインゴールとして選択する。
また `change` と同様に、定義的に等しい別の型でゴールを書き換える効果もある。

---

## ゴールパターン

**適用前（複数ゴール）**
```lean
-- ゴール 1: ⊢ A
-- ゴール 2: ⊢ B
```

**適用後（`show B`）**
```lean
-- ゴール 1: ⊢ B  （メインに移動）
-- ゴール 2: ⊢ A
```

---

## 構文

```lean
show P
```

`P` はいずれかのゴールの型と定義的に等しくなければならない。

---

## 必須 import

Lean4 core 組み込みのため不要。

---

## オプション一覧

| 構文 | 意味 |
|---|---|
| `show P` | 型 `P` にユニファイできるゴールをメインゴールにする |

特になし。`show` は単一構文のみ。

---

## Pros（使うべき場面）

- 複数ゴールがある場合に、処理したいゴールを明示的に選択できる
- ゴールの型を明示するため、証明の可読性が向上する
- `change` と同様に定義的に等しい型への書き換えができる

---

## Cons（注意・失敗ケース）

- 指定した型がどのゴールともユニファイできない場合はエラーになる
- ゴールが 1 つしかない場合はドキュメンテーション目的以外のメリットが少ない
- `change` とほぼ同等であり、機能的な差は小さい

---

## コードサンプル

### サンプル 1: ゴールの型を明示

```lean
example : 1 + 1 = 2 := by
  -- ⊢ 1 + 1 = 2
  show 1 + 1 = 2
  -- ⊢ 1 + 1 = 2（型の明示、ドキュメンテーション目的）
  rfl
```

### サンプル 2: 複数ゴールからの選択

```lean
example : True ∧ (1 = 1) := by
  constructor
  -- ゴール 1: ⊢ True
  -- ゴール 2: ⊢ 1 = 1
  show 1 = 1
  -- ゴール 1: ⊢ 1 = 1（メインに移動）
  -- ゴール 2: ⊢ True
  · rfl
  · trivial
```

### サンプル 3: 定義的等価を利用した書き換え

```lean
example : Nat.succ 0 + Nat.succ 0 = 2 := by
  -- ⊢ Nat.succ 0 + Nat.succ 0 = 2
  show 1 + 1 = 2
  -- ⊢ 1 + 1 = 2
  rfl
```

## 関連タクティク

| タクティク | 関係 | 使い分け |
|---|---|---|
| `change` | 類似 | `change` もゴールの型を書き換える。`show` はゴール選択機能がある |
| `swap` | ゴール操作 | `swap` は先頭 2 つのゴールを入れ替える。`show` は型で選択 |
| `pick_goal` | ゴール操作 | 番号でゴールを選択する。`show` は型で選択する |
| `suffices` | 中間ゴール | 新しい中間命題を導入する。`show` は既存ゴールの選択のみ |
