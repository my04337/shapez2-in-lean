# Session Efficiency Follow-up Plan (C/E/H)

> 作成日: 2026-04-18
> 最終更新: 2026-04-18
> 対象: セッション効率改善の追加検討（提案 C/E/H）

---

## 目的

A/B/D/F/G は即時適用済みのため、本計画では追加検討が必要な以下 3 項目を評価し、導入可否を決定する。

- C: system prompt 軽量化（低頻度規則の外部参照化）
- E: 小幅 read の自動抑制（短幅 2 回で 100 行以上へ強制）
- H: todo 更新頻度の追加間引き（開始/完了中心の運用強化）

---

## 検討スコープ

### C. system prompt 軽量化

評価観点:

- 常時同梱される規則のうち、低頻度参照セクションの候補抽出
- 外部化後の発動精度（参照漏れ・逸脱率）の変化
- 入力トークン削減効果（中央値・上位 90 パーセンタイル）

完了条件:

- 外部化候補セクションと移動先の一覧を確定
- 「精度低下リスク中以下」の候補のみを採用案として提示

### E. 小幅 read 自動抑制

評価観点:

- `read_file` 50 行未満呼び出しの発生頻度
- 100 行以上へ強制した場合の再読回数・探索時間の変化
- docs タスクと Lean タスクでの副作用差（過読みによるノイズ）

完了条件:

- 強制条件（回数・対象・除外条件）を定量的に確定
- 既存ルールとの競合がないことを確認

### H. todo 更新頻度の追加間引き

評価観点:

- 現在の `manage_todo_list` 呼び出し密度
- 進捗可視性を維持しつつ削減できる更新回数
- ブロッカー発生時の例外運用ルール

完了条件:

- タスク種別別の上限回数を確定
- 可視性低下がないことをレビューで確認

## 判定基準

- C 採用基準: 入力トークン中央値 10% 以上削減、かつ参照漏れによる逸脱増加が軽微
- E 採用基準: 小幅 read 回数 30% 以上削減、かつ探索失敗率の悪化なし
- H 採用基準: todo 更新回数 40% 以上削減、かつ進捗報告の欠落なし

	- $13,824,057 \times 0.30 = 4,147,217$ tokens
	- 全体比で **約 3.35%** 削減見込み
