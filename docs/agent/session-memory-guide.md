# セッションメモリ運用ガイド

> **位置づけ**: `AGENTS.md` の「セッションメモリの作成義務」セクションの詳細リファレンス。
> コアルールは `AGENTS.md` に記載。このドキュメントはテンプレート・詳細ルールを提供する。

---

## テンプレート

1 セッションにつき原則 **2 ファイル以内** に統合する。断片化（5 ファイル以上）はコンテキスト圧縮後の復元コストが増大するため禁止。各ファイルのサイズは **5KB 以内** を目安とする。探索結果の生のコードスニペットや全行番号リストは省き、要点のみ箇条書きにまとめること。

```markdown
# /memories/session/progress.md — 必須: TODO + 現在の状態
## Status: errors=X, sorry=Y (verified build at HH:MM)
## TODO
- [x] 完了タスク
- [ ] 未完了タスク
## Line Numbers (Explore 結果)
| 項目 | 行 |
|---|---|
## Next Session Plan
```

```markdown
# /memories/session/technical-notes.md — 必要に応じて: 戦略・REPL検証済みコード
## Strategy
## Informal Sketches (sorry 別非形式スケッチ)
<!-- 各 sorry に対して、タクティクに取り掛かる前に自然言語で 1〜2 行の証明方針を記録する -->
| sorry 位置 | 非形式スケッチ |
|---|---|
| 例: Gravity.lean:1234 | 「foldl の帰納法で、各ステップで immBelow が保存されることを示す」 |
## REPL Verified Code
<!-- 証明計画の補題は必ず検証状態を記入（verified=✅ / ❌ 未検証） -->
| 補題名 | 検証状態 | 検証コマンド |
|---|---|---|
| 例: foldl_obstacle_preserved | ❌ 未検証 | — |
## Error Patterns (再発防止)
```

圧縮前や作業の区切りで新ファイルを作りたくなった場合は、**既存ファイルにセクションを追記**すること。

## 更新方法

- 既存セッションメモリの更新は `str_replace` / `insert` を使う（部分更新）
- 全面書き換えが必要な場合も `str_replace` で既存内容を一括置換する
- `delete` はセッション終了時のクリーンアップでのみ使う

> **理由**: `delete → create` サイクルはコンテキスト圧縮と重なるとメモリロスになる。部分更新のみ保つことで中間状態も復元可能なまま保たれる。

## 即時作成が必須のタイミング

- **Wave / 多段証明を開始した直後**（計画の要約・対象ファイル・行範囲を直ちに保存）
- **`build.ps1` の出力を受信した直後**（ビルドを含む多段作業の場合、出力確認後 3 ツール呼び出し目までに sorry 一覧・ターゲット・ステータスを保存）
- **コンテキスト圧縮（context compaction）が間もなく発生しそうな時**（会話が長くなったら予防的に保存）
- **分析・計画セッション（コード変更なし）でも、結論をセッションメモリに記録する**（例: 「X は Y に影響なし」「提案 Z を採用」等。会話圧縮で分析結論が失われるのを防ぐ）

## 通常の作成タイミング

| タイミング | 保存内容 |
|---|---|
| 大規模探索完了後 | ファイル構造の把握結果、import チェーン、対象の行範囲 |
| 証明戦略が確定した時点 | 使用する補題リスト、証明の骨格、未解決ゴール一覧 |
| ビルドエラー修正後 | エラー原因と修正パターン（同じ間違いの再発防止） |
| sorry 1件解消ごと | 解消した sorry とその証明手法の要約 |
| Lean コード変更後（sorry 行番号が変化した時） | `progress.md` の Sorry Locations テーブルを実際の行番号に同期 |

## sorry 行番号の記録ルール

sorry の位置をセッションメモリに記録するときは、以下の **2 種類の行番号** を必ず区別して記録すること。

| 種類 | 内容 | 取得方法 |
|---|---|---|
| **ビルド診断行** | `[SORRY] File.lean:L:C` に表示される行（定理宣言の先頭行） | `build.ps1` の `[SORRY]` 出力 |
| **sorry キーワード行** | ソースコード中で `sorry` とタイプされている行 | `grep_search "sorry" inFile.lean` |

**禁止**: 会話サマリー・記憶・前セッションのメモから行番号を転記すること。コード変更で行番号はずれるため、常に `grep_search` か `read_file` で確認した値のみを使う。

## 圧縮前の同期義務（PreCompact チェックリスト）

1. `manage_todo_list` で全 todo の状態を最新に更新する（完了済みは completed に）
2. セッションメモリの進捗テーブルを最新化する（todo と矛盾がないように）
3. 作業中の未コミット変更があればファイルパス・行番号をメモに含める

## セッション終了時のクリーンアップ（必須）

ユーザーが「終わり」「コミットして」等で作業完了を宣言したら、以下を実施する。

1. 完了したタスクのセッションメモリを **削除**（`memory delete`）する
2. 次セッションに引き継ぐ内容のみ残す（残存 sorry・未完了タスク・次回計画）
3. `/memories/session/` が 2 ファイル以内に収まっていることを確認する

> **理由**: 完了済みの作業メモが蓄積すると断片化が進み、次セッションのコンテキスト復元コストが増大する。
